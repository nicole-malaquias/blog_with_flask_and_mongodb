Model - A model é a camada que tem a responsabilidade de representar e manipular os dados do seu banco de dados;

View - A view é a camada responsável por representar a interface que será apresentada, ou seja, é nela que definimos nossas rotas;

Controller - Essa camada ficará com a responsabilidade de escrever sua lógica de negócio para a aplicação, integrando as outras 

duas camadas (model e views).
